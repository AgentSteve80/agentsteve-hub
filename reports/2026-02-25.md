## 1. вҡ пёҸ CVE-2026-27597 вҖ” Enclave AI Sandbox CVSS 10.0 RCE (Sandbox Escape)
**Source:** TheHackerWire / OffSeq Threat Radar В· **Published:** Feb 25, 2026
**Relevance:** вӯҗвӯҗвӯҗвӯҗвӯҗ (5/5)
**Tags:** `#security` `#ai` `#rce` `#sandbox` `#agents` `#patch-now`

**CVE-2026-27597** is a maximum-severity (CVSS 10.0) remote code execution vulnerability in **Enclave** вҖ” a JavaScript sandbox built specifically to safely execute AI agent code. The flaw (CWE-94: Improper Control of Code Generation) allows attackers to escape sandbox boundaries entirely in versions prior to **2.11.1**. Published today by `agentfront/enclave` on GitHub. Exploit path: an attacker who can influence the code submitted to the sandbox (e.g., via a prompt injection chain) can achieve full host-level code execution.

**So what:** This is the nightmare scenario for any system that runs LLM-generated code in a sandbox вҖ” including agentic CI, OpenClaw sub-agents, and any future Pal app automation. If any part of our stack uses `agentfront/enclave` (check `npm list | grep enclave`), update to 2.11.1 immediately. More broadly: this confirms that JS sandboxes for AI code are a prime CVE target. Any agent that executes code needs layered isolation (VM + namespace + seccomp), not just a sandbox library.

## 2. вҡ пёҸ CVE-2026-25639 вҖ” Axios DoS in Node.js Apps, CVSS 7.5
**Source:** Indusface / Security Boulevard В· **Published:** Feb 24вҖ“25, 2026
**Relevance:** вӯҗвӯҗвӯҗвӯҗвӯҗ (5/5)
**Tags:** `#security` `#nodejs` `#axios` `#dos` `#patch-now` `#stack`

**CVE-2026-25639** is a remotely triggerable **denial-of-service** vulnerability in **Axios** вҖ” the most popular HTTP client for Node.js вҖ” caused by improper handling of crafted configuration objects. CVSS 7.5 (High), no auth required, network-accessible. Patched in **Axios 1.13.5**. Any Node.js service that accepts user-controlled input passed into Axios config objects is vulnerable to process crash.

**So what:** Axios is in virtually every Node.js project including Next.js, Convex functions, and any tool that makes HTTP requests. Check `package.json` in `mission-control`, `pal-web`, and any Convex HTTP action that uses Axios. Run `npm audit` вҖ” if Axios appears < 1.13.5, update now. This one has no authentication requirement, meaning it's trivially exploitable by anyone who can send a crafted HTTP request to an exposed endpoint.

OWASP released a dedicated **Top 10 for Agentic Applications** вҖ” developed with 100+ industry security experts вҖ” covering the unique attack surface of autonomous AI agents. **So what:** This is the authoritative checklist for our OpenClaw deployment. Map every OWASP Agentic Top 10 item to OpenClaw's current mitigations and log them in the Mission Control Security page. Immediate priorities: (1) ASI01 вҖ” our system prompt boundaries are the first line; audit them against the OWASP criteria. (2) Tool misuse вҖ” review which tools each agent can access and enforce least-privilege per the new pal/main agent split. (3) Privilege escalation вҖ” verify agent вҶ’ gateway вҶ’ host boundaries are enforced. Full list: <https://securitywall.co/blog/owasp-top-10-for-agentic-ai-2026>

## 7. OliveTin вҖ” Critical Unauthenticated RCE (Web UI for Shell Commands)
**Source:** TheHackerWire В· **Published:** Feb 25, 2026
**Relevance:** вӯҗвӯҗвӯҗ (3/5)
**Tags:** `#security` `#selfhosted` `#rce` `#patch-now`

**OliveTin** (a self-hosted web UI that lets you run shell commands from a browser) has a critical unauthenticated RCE vulnerability вҖ” CVE-2026-27626. The default config has `authType: none` and registration enabled, effectively making it open to anyone who can reach the port. If you run OliveTin anywhere in your self-hosted stack, treat it as fully compromised until patched.

**So what:** We don't run OliveTin, but it's a reminder that any tool that executes shell commands via a web interface is an extraordinarily high-value target. Mission Control's security audit endpoint (`/api/security/run`) should similarly be locked down to authenticated admin requests only вҖ” worth auditing.
