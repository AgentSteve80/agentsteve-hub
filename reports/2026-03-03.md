## 1. 🔴 StegaBin — DPRK Famous Chollima Hides RAT C2 in Pastebin Essays via 26 Malicious npm Packages
**Source:** Socket.dev / The Hacker News · **Published:** March 2, 2026
**Relevance:** ⭐⭐⭐⭐⭐ (5/5) — **DIRECT: every `npm install` of a typosquatted package in our repos is this attack**
**Tags:** `#security` `#npm` `#supply-chain` `#DPRK` `#RAT` `#typosquatting` `#action-required`

Socket's threat research team (Philipp Burckhardt and Peter van der Zee) disclosed **StegaBin** — a coordinated DPRK supply chain campaign publishing **26 malicious npm packages** on February 25–26, 2026, each under individual throwaway accounts. The packages masquerade as developer utilities. The technical innovation is the C2 delivery: rather than hardcoding infrastructure addresses, the loader reads **three Pastebin "computer science essays"** and extracts C2 URLs by reading characters at evenly-spaced positions — character-level steganography. The actual C2 infrastructure is split across **31 Vercel deployments**. Once installed, it drops a **9-module credential stealer and cross-platform RAT** that targets: SSH keys, shell history, environment variables (`.env` files), browser-stored credentials, and VS Code settings (where API keys are often stored). All 26 packages share a single malicious loader at `vendor/scrypt-js/version.js` (SHA256: `da1775d0fbe99fbc35b6f0b4a3a3cb84da3ca1b2c1bbac0842317f6f804e30a4`). The campaign is attributed to **Famous Chollima** (aka Contagious Interview) — the same North Korean threat cluster behind the 2024 npm supply chain attack series. C2 live at `103[.]106[.]67[.]63:1244`.

**So what:** Four direct action items. (1) **Check our `package.json` files** for typosquatted names — Famous Chollima specifically targets packages with names like `react-native-*`, `expo-*`, `convex-*`, and `clerk-*` variants. Run `npm ls 2>&1 | grep -v npm` on all repos and compare against the Socket published list. (2) **Never run `npm install` from a pasted command you haven't verified** — especially from a GitHub Issue comment, a Discord message, or a web tutorial that asks you to run a specific install command. The Contagious Interview campaign's initial vector is "fake job interview technical tasks" that include malicious package installs. (3) **The 9-module stealer targets `.env` files specifically.** Our secrets are in `~/.openclaw/secrets/convex.env`, `.env.local` in project dirs, and `~/.expo/state.json` — exactly what this stealer targets. If you ever run an unverified `npm install` or `npx` command, those files are at risk. (4) **The C2 uses Vercel deployments** — this means standard IP/domain blocklists won't catch it. Legitimate Vercel URLs are whitelisted by most corporate firewalls.

## 3. 📉 AI "Vibe Coding" Is Triggering an Open Source Crisis — Maintainers Are Closing Projects
**Source:** InfoQ / The Register / Dataconomy · **Published:** January 26 – March 3, 2026
**Relevance:** ⭐⭐⭐⭐ (4/5) — **Pattern: we're heavy AI coding users — OSS deps we rely on are at risk**
**Tags:** `#AI` `#open-source` `#ecosystem` `#vibe-coding` `#maintainers` `#tech-debt` `#community`

An economics paper (covered by InfoQ and The Register, February 2026) models the systemic risk of "vibe coding" — where AI agents select and assemble open-source packages without developers reading documentation, reporting bugs, or engaging with maintainers. The economic model shows a **negative feedback loop**: AI handles package selection → fewer humans read docs → fewer bug reports → fewer community interactions → maintainer income and recognition drop → maintainers burn out and close projects. Real-world evidence reported alongside the paper: `curl` (the foundational HTTP library in every JS runtime, including Node.js) received **20 AI-generated security reports in 21 days**, all invalid, requiring manual review time from Daniel Stenberg. Multiple projects have started closing GitHub issues from accounts that appear to be AI-assisted with no human verification. GitHub Blog simultaneously published that "projects likely to succeed in 2026 will use AI **in maintainership**" — automating triage, labeling, and duplicate detection — rather than just using AI for coding.

**So what:** Our entire stack runs on OSS — React Native (Meta), Expo (Expo), Convex, Clerk, Next.js, supabase-js, and hundreds of transitive dependencies. If AI-flooding erodes maintainership on any of these, security patches slow down, CVE response degrades, and APIs break without notice. Three responses: (1) **When we use an OSS package and find a real bug, file the issue manually with a human-quality report** — this is both good citizenship and helps ensure the package remains maintained. (2) **Pin deps to exact versions in production** (`"convex": "1.32.0"` not `"^1.32.0"`) — reduces exposure to sudden breakage if a maintainer rage-quits. (3) The GitHub Blog angle is actionable: **watch for OpenClaw adopting AI-as-maintainer features** — automated issue triage, PR labeling, and duplicate detection are exactly the kind of community infrastructure that would justify the OpenClaw 230k-star trajectory as something more than just a coding tool.
