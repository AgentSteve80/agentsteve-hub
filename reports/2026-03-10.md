### 2. 🚨 CVE-2025-55182 "React2Shell" — RCE in React Server Components, 8M Exploit Attempts
**Relevance: 10/10** | `#security` `#react` `#nextjs` `#rce` `#critical`

A critical flaw in React's Flight protocol (**CVE-2025-55182**, "React2Shell") allows attackers to run arbitrary code on servers using React Server Components. 8 million logged exploit attempts by early January 2026 per GreyNoise — this is bulk exploitation at scale. DPRK linked actors used it to deploy EtherRAT. Active exploits abuse NGINX misconfigurations for traffic hijacking. Affects Next.js 14.3.0+, Next.js 15, and Next.js 16 (Dynatrace detection query available).

**So what:** **This hits us directly.** Mission Control is Next.js 15+ with React Server Components. Pal Web too. Priority action: run Dynatrace's detection query (or equivalent) to check if your Next.js version and RSC usage is in the affected range. Patch to latest Next.js immediately. Check that RSC endpoints don't accept unsanitized serialized input. This is actively being exploited at scale — not a future concern.

🔗 https://www.dynatrace.com/news/blog/cve-2025-55182-react2shell-critical-vulnerability-what-it-is-and-what-to-do/
🔗 https://hackaday.com/2026/03/06/this-week-in-security-getting-back-up-to-speed/

### 7. 🛠️ GitHub Blog: Agentic Workflow Security Architecture — How to Do It Right
**Relevance: 8/10** | `#security` `#agents` `#github` `#cicd` `#devtools`

GitHub published a deep-dive on the security architecture of their agentic workflows today — covering permission scoping, safe-outputs (capping how many issues/PRs an agent can create), tool-level policy, and the trust model for agents that read public GitHub content (issues, comments, PRs).
