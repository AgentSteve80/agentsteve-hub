### 5. 🚨 S3 Bucket Squatting Is Finally Dead — Security Win
**Relevance: 8/10** | `#security` `#aws` `#cloud` `#infrastructure`

HN #2 today (176 pts): AWS permanently closed the bucket squatting vulnerability — where a malicious actor could register an S3 bucket with the same name as a previously deleted bucket to hijack traffic. A 10+ year vulnerability finally patched.

**So what:** If any of our infrastructure references S3 bucket URLs (CloudFront, SiteSignal data pipelines, file uploads), this was previously a re-registration risk after deletion. That risk is now gone. Good AWS hygiene still applies: never delete and re-create buckets with the same name if they serve production traffic.

🔗 https://onecloudplease.com/blog/bucketsquatting-is-finally-dead
