## 1. ⚠️ CVE-2026-26960 — node-tar Hardlink Escape: Arbitrary File Write in Every npm Install
**Source:** GitLab Advisory / Windows Forum / OffSeq Threat Radar · **Published:** Feb 25–26, 2026
**Relevance:** ⭐⭐⭐⭐⭐ (5/5)
**Tags:** `#security` `#nodejs` `#npm` `#supply-chain` `#patch-now` `#stack`

**CVE-2026-26960** is a high-severity path traversal flaw in **node-tar** (`isaacs/node-tar`) — the package used by npm, yarn, and pnpm to extract archive files during `npm install`. A crafted tar archive can create a hardlink inside the extraction directory pointing to a file *outside* the extraction root, allowing arbitrary file read and write as the extracting user. Affects versions ≤ 7.5.7. **Fixed in node-tar 7.5.8**. Every Node.js project that runs `npm install` on untrusted or CI-fetched packages is in scope.

**So what:** This hits every project in our stack — Mission Control, Pal Web, Pal Field App. A supply chain attacker who compromises a dependency upstream could include a malicious archive that writes files outside the project directory when installed. Run `npm audit` on all three projects now; ensure node-tar is at 7.5.8+. Also check GitHub Actions runners — if CI does `npm install`, a compromised transitive dep could write to the runner filesystem. Related: pair this with `npm audit --audit-level=critical` in all CI pipelines.

## 2. ⚠️ CVE-2026-27606 — Rollup Bundler Path Traversal: Affects Vite, Next.js Build Chains
**Source:** Dev.to / CVEReports / Vulners · **Published:** Feb 26, 2026
**Relevance:** ⭐⭐⭐⭐⭐ (5/5)
**Tags:** `#security` `#rollup` `#nextjs` `#vite` `#build-toolchain` `#patch-now` `#stack`

**CVE-2026-27606** is an arbitrary file write vulnerability in **Rollup** — the bundler that powers Vite, and transitively Next.js and Expo's web builds. Due to a fragile regex-based blacklist for output filename sanitization (instead of proper path normalization), attackers can bypass the sanitization and write bundle output files to arbitrary locations on the filesystem. Affects Rollup < 2.80.0, < 3.30.0, and < 4.59.0. Patched in the latest versions of each branch.

**So what:** Mission Control and Pal Web both run on Next.js which uses Rollup under the hood via Webpack/Turbopack. This is exploitable in CI environments where build output paths may be user-influenced (e.g., via env vars or config injection from a compromised upstream dep). Run `npm list rollup` in all projects; update to a patched version. Also check if Vercel's build environment pins Rollup versions — if so, confirm they've patched their build images.

## 5. 💀 DarkCloud Infostealer: 300k+ ChatGPT Credentials Stolen, AI Platforms Now Prime Target
**Source:** GBHackers / IBM X-Force · **Published:** Feb 26, 2026
**Relevance:** ⭐⭐⭐⭐ (4/5)
**Tags:** `#security` `#infostealer` `#credentials` `#ai` `#threat-intel`

The **DarkCloud** infostealer is escalating as a major enterprise threat. According to IBM's X-Force analysis and GBHackers reporting, infostealer malware led to the exposure of **over 300,000 ChatGPT credentials** in 2025 — signaling that AI platforms are now high-value credential targets on par with core enterprise SaaS. DarkCloud is commercialized MaaS (Malware-as-a-Service), available cheaply on dark web forums, requiring zero technical skill to operate.

**So what:** Our stack uses Anthropic API keys, Clerk auth tokens, Convex deploy keys, and Vercel tokens. If a developer's machine is compromised by an infostealer, ALL of these rotate to an attacker. Immediate action: (1) ensure all API keys are stored in `.env.local` (not hardcoded), (2) rotate any key that has ever touched a browser, clipboard, or chat session, (3) enable Clerk's suspicious sign-in detection for the Pal Low Voltage org. Also: Claude/ChatGPT credentials are now worth stealing — be careful what you paste into chat windows.

## 7. 🔐 Proofpoint: Full Lifecycle of Account Compromise in the Agentic Workspace
**Source:** Proofpoint Blog · **Published:** Feb 25, 2026
**Relevance:** ⭐⭐⭐⭐ (4/5)
**Tags:** `#security` `#agents` `#account-takeover` `#identity` `#openClaw`

Proofpoint published a detailed lifecycle analysis of **account compromise in agentic workspaces** — specifically how attackers exploit the trust relationships between users, their AI agents, and the services agents are authorized to access. Key stages: initial access via phishing or infostealer → session token hijacking → agent impersonation → lateral pivot through agent tool permissions (email, calendar, file storage, code repos). The attack is harder to detect because agents perform actions continuously at scale — attacker actions blend in.

**So what:** This maps directly to our setup. If OpenClaw's session credentials were compromised, an attacker could silently use the gateway to access Andrew's files, send Discord messages, or trigger Convex mutations — all appearing as legitimate agent activity. Mitigations: (1) gateway auth token rotation (done — we rotated last week), (2) ensure OpenClaw's tool policies log all external API calls, (3) treat the OWASP Agentic Top 10 as a firewall checklist for what our agents can access. The Pal/Claw agent split we set up yesterday is actually good practice here — least-privilege per agent reduces blast radius.
