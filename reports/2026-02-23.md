### 1. 🔴 Supply Chain Attack: Cline CLI 2.3.0 Silently Installed OpenClaw on Developer Machines
**Relevance: 10/10** `#security` `#supply-chain` `#openclaw` `#npm` `#immediate`
**Links:** [The Hacker News](https://thehackernews.com/2026/02/cline-cli-230-supply-chain-attack.html) · [The Register](https://www.theregister.com/2026/02/20/openclaw_snuck_into_cline_package/) · [Dark Reading](https://www.darkreading.com/application-security/supply-chain-attack-openclaw-cline-users) · [Endor Labs](https://www.endorlabs.com/learn/supply-chain-attack-targeting-cline-installs-openclaw)

On February 17, a compromised npm publish token was used to push `cline@2.3.0` — a malicious version of the popular Cline AI coding assistant. The package **silently installed OpenClaw globally on any developer machine that ran `npm install cline`**. A prior prompt injection in Cline's GitHub Actions workflow (Feb 9) allowed the attacker to compromise production releases on both the VS Code Marketplace and OpenVSX. The malicious package has since been pulled, but the window of exposure was approximately 3 days.

**So what:** This is maximally relevant to your setup — OpenClaw is your AI layer, and it just got used as the payload in a supply chain attack. This confirms two things: (1) OpenClaw's popularity makes it an attractive malware vector, and (2) your decision to install zero ClawHub skills is exactly right. If you install OpenClaw from an untrusted source or update without verifying, you may be running someone else's agent. Stick to the official `npm -g openclaw` install path and pin your version. The fact that the attacker chose OpenClaw as the payload suggests it has root/system-level access on install — which is why your hardened VM matters.
