### 4. тЪая╕П CVE-2026-30851 (CVSS 8.1) тАФ Caddy `forward_auth` Identity Injection / Privilege Escalation
**Relevance: 9/10** | `#security` `#caddy` `#auth` `#cve` `#infrastructure`

Critical auth bypass in **Caddy's `forward_auth` directive with `copy_headers`**: conditional header-set operations only fire when the upstream auth service returns the header тАФ if it doesn't return a header, client-supplied values pass through unchanged. Attackers can inject identity headers, bypass access control, and escalate privileges. Affects Caddy 2.10.0 through 2.11.1. **Patched in v2.11.2.**

**So what:** Our OpenClaw gateway runs behind Caddy (Tailscale serve uses Caddy internally). Check your Caddy version: `caddy version`. If you use `forward_auth` with `copy_headers` in any Caddyfile config, patch immediately. Even if not directly using that directive, upgrade to 2.11.2 as a precaution.

ЁЯФЧ https://advisories.gitlab.com/pkg/golang/github.com/caddyserver/caddy/v2/modules/caddyhttp/reverseproxy/CVE-2026-30851/
ЁЯФЧ https://www.thehackerwire.com/vulnerability/CVE-2026-30851/
