**Source:** The Hacker News / SOCRadar / ESecurity Planet · **Published:** Feb 25–27, 2026
**Relevance:** ⭐⭐⭐⭐ (4/5)
**Tags:** `#security` `#CVE` `#cisco` `#zero-day` `#CVSS10` `#network` `#active-exploitation`

**CVE-2026-20127** in Cisco Catalyst SD-WAN: an unauthenticated remote attacker sends a single crafted HTTP request to the management interface and obtains full administrative privileges. CVSS 10.0. The twist: Cisco Talos confirmed exploitation activity by threat actor "**UAT-8616**" dating back to **at least 2023** — meaning this vulnerability was silently weaponized for years before disclosure. Confirmed as actively exploited in the wild as of February 25, 2026. Indicators: unauthorized control-connection peering events in logs. Patch is available.

**So what:** This is the nightmare scenario that the DoD's air-gapped network requirement (Edition 10) is explicitly trying to prevent — a zero-day exploited silently for 3 years before the vendor even knew. For our stack: we don't run Cisco SD-WAN, but the pattern is instructive. Whenever a network appliance sits between the internet and internal infrastructure (routers, VPN concentrators, any management interface), it needs to be: (1) not internet-exposed, (2) on a patching schedule, (3) monitored for anomalous auth events. Our OpenClaw gateway on port 7443 — confirm it is NOT internet-accessible. SecureClaw audit should include a port exposure check on that service.

## 2. 🌐 Chrome CVE-2026-2441 (CVSS 8.8) — CSS Use-After-Free RCE, Patched Feb 13, Actively Exploited
**Source:** The Hacker News / Orca Security / Malwarebytes / Purple Ops · **Published:** Feb 13–24, 2026
**Relevance:** ⭐⭐⭐⭐⭐ (5/5) — **AFFECTS ALL CHROMIUM-BASED BROWSERS + EXPO WEBVIEW**
**Tags:** `#security` `#CVE` `#chrome` `#RCE` `#webview` `#expo` `#react-native` `#action-required`

A CSS use-after-free vulnerability in Chrome's font handling engine. An attacker serves a malicious webpage → browser executes arbitrary code in the sandboxed renderer → can escape to OS. Affects **all Chromium-based browsers**: Chrome, Edge, Brave, Opera. Patch shipped February 13 (5 days after report by Shaheen Fazim on Feb 11). Actively exploited before the patch. CVSS 8.8.

**So what:** The critical detail for our stack: **React Native on Android uses Chromium's WebView** for any web content rendered via `<WebView>` component. If `pal-field-app` renders any web content (the email preview in the Gmail jobs pipeline, any PDF preview, the map view via Leaflet WebView), that WebView inherits the Android system's Chromium version — which updates independently from the app. The practical check: (1) does pal-field-app render any external URLs in a WebView? Audit `app/(tech)/settings/gmail-jobs.tsx` — if it previews email HTML, that's an attack surface. (2) iOS uses WKWebView (WebKit), not Chromium — not affected by this CVE. (3) **macOS Tahoe** (Andrew's dev machine) — update Chrome now if it hasn't auto-updated past the Feb 13 patch.

## 4. 🪱 Ivanti EPMM "Sleeper" Webshells — European Commission Breached, Dutch DPA Compromised
**Source:** Help Net Security / Unit 42 (Palo Alto) / Tenable / GreyNoise · **Published:** Feb 11–25, 2026
**Relevance:** ⭐⭐⭐⭐ (4/5)
**Tags:** `#security` `#CVE` `#ivanti` `#webshell` `#persistence` `#MDM` `#supply-chain`

**CVE-2026-1281 + CVE-2026-1340** in Ivanti EPMM (mobile device management). Root cause: Bash arithmetic expansion in EPMM's file delivery mechanism → unauthenticated RCE. Both exploited as zero-days. Patches shipped Feb 4. The disturbing new development (Feb 11): researchers found **"sleeper" webshells** — dormant backdoors installed before the patch that survive updates and sit idle until activated. Confirmed victims: **European Commission mobile device management platform**, **Dutch Data Protection Authority**, **Council for the Judiciary (Netherlands)**. All breached via CVE-2026-1281 on or before Jan 29.

**So what:** Two lessons relevant to Pal: (1) **MDM is a high-value target.** Our devices (the Mac Mini M4 running OpenClaw, Andrew's iPhone with the EAS build) are not on an enterprise MDM — that's good, it removes this attack surface. If Pal Low Voltage ever adopts an MDM for field tech iPhones (common in construction), choose a provider with a track record of fast CVE response. Ivanti has had 3+ critical EPMM zero-days in 18 months. (2) **Sleeper webshell pattern**: patch application does not guarantee cleanup. After patching any critical RCE, the correct response is to audit for persistence mechanisms (webshells, cron jobs, new user accounts) before assuming clean. This applies to any server we run.
