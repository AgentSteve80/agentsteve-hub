### 1. 🚨 CNCERT Advisory: OpenClaw Prompt Injection & Data Exfiltration Risks
**Relevance: 10/10** | `#openclaw` `#security` `#prompt-injection` `#critical`

China's National Computer Network Emergency Response Technical Team (CNCERT) issued a public warning about security vulnerabilities in OpenClaw, flagging "inherently weak default security configurations" that enable prompt injection attacks and sensitive data exfiltration through agentic workflows. The advisory specifically names OpenClaw's tool access (files, shell, messaging) combined with insufficient input sanitization as the attack surface.

**So what:** This is us. Our posture: all external content is already wrapped as `UNTRUSTED`, gateway is not publicly exposed, shell access is scoped. Still — this is a good prompt to audit our webhook and channel ingestion surfaces. Anything that flows into the agent from an external source (Discord, GitHub webhooks, email) is a potential injection vector. Running the healthcheck skill and reviewing MCP connections is the right response. The CNCERT advisory also signals that nation-state actors are studying OpenClaw deployments for weaknesses — don't relax the defaults.

### 2. 🚨 Malicious npm Package `@openclaw-ai/openclawai` Deploys RAT (178+ Downloads)
**Relevance: 10/10** | `#openclaw` `#supply-chain` `#npm` `#critical`

A malicious npm package published March 3, 2026 impersonates the OpenClaw installer, deploys a remote access trojan, and steals macOS keychain credentials. 178+ downloads, package is still live. The legitimate package is `openclaw` (no scope). The fake uses `@openclaw-ai/openclawai`.

**So what:** Direct impersonation of our toolchain. If you ever reinstall or update OpenClaw on the Mac Mini, use only: `npm install -g openclaw`. Never `@openclaw-ai/openclawai` or any scoped variant. If you installed anything recently from an unfamiliar source, run `npm ls -g --depth=0` and audit the global package list. Also worth checking CI workflows — if any pipeline does `npm install openclaw`, verify it's the unscoped package.

### 3. CrackArmor: 9 AppArmor Kernel Flaws — Root Escalation + Container Escape (Ubuntu)
**Relevance: 9/10** | `#linux` `#ubuntu` `#kernel` `#apparmor` `#critical`

Qualys TRU disclosed 9 "confused deputy" vulnerabilities in the Linux kernel's AppArmor module (CrackArmor). Unprivileged users can circumvent kernel security, escalate to root, and break container isolation.

**So what:** We're running kernel `6.8.0-101-generic` with AppArmor active (105 profiles loaded, 10 in enforce mode). A newer kernel is available: `6.8.0-106.106` — that's 5 patch increments ahead and likely contains the CrackArmor fixes. **Action: `sudo apt-get install --only-upgrade linux-generic && sudo reboot`** — this is a kernel security update, not an OS upgrade. Safe to do. Want me to run this?

### 4. AppsFlyer Web SDK Hijacked — Crypto-Stealer JavaScript Spread via Supply Chain
**Relevance: 9/10** | `#supply-chain` `#sdk` `#javascript` `#security`

The AppsFlyer Web SDK was temporarily injected with malicious JavaScript this week to steal cryptocurrency from end users. AppsFlyer is widely used for mobile app attribution — if you have it in a web app, your users' browsers executed attacker code.

**So what:** Pal Web and Mission Control don't currently use AppsFlyer. But this is a strong signal that **any third-party analytics/attribution SDK is a supply chain attack surface**. If AppsFlyer is ever added to pal-web or the field app for install attribution, gate it behind a CSP nonce and consider self-hosting the SDK or using a subresource integrity hash.

### 6. CISA KEV: n8n RCE CVE-2025-68613 (CVSS 9.9) — 24,700 Exposed Instances
**Relevance: 8/10** | `#automation` `#rce` `#cisa` `#n8n`

CVE-2025-68613 (expression injection → RCE) in n8n added to CISA Known Exploited Vulnerabilities. Two additional critical n8n CVEs (CVSS 9.4, 9.5) allow sandbox escape and unauthenticated credential exposure. Over 24,700 public n8n instances remain unpatched.

### 7. GlassWorm: 72 Open VSX Extensions Compromised via Transitive Dependency Abuse
**Relevance: 8/10** | `#supply-chain` `#vscode` `#extensions` `#developer-security`

Attackers abused `extensionPack` and `extensionDependencies` metadata in the Open VSX registry to turn innocent extensions into transitive malware loaders. 72 extensions affected in this wave.

**So what:** If you use VS Code, Cursor, or Windsurf with Open VSX extensions, audit your installed extensions. The transitive abuse means extensions you trust can pull in malicious code as "dependencies." Check `~/.vscode/extensions` or equivalent and verify publishers for anything you installed in the last 30 days.

### 8. Five Malicious Rust Crates Steal .env Files from CI/CD Pipelines
**Relevance: 8/10** | `#supply-chain` `#rust` `#cicd` `#secrets`

Five fake crates on crates.io (chrono_anchor, dnp3times, time_calibrator, time_calibrators, time-sync) impersonate time utilities and exfiltrate `.env` files to attacker servers. An AI bot automated their publishing and evasion. Published late-Feb to early-March 2026.

**So what:** `.env.local` in mission-control and pal-web are the files at risk. Verify: (1) `.env.local` is in `.gitignore` in all repos, (2) CI env vars are injected at runtime by Vercel/GitHub Actions, never read from committed files. Run `git log --all --full-history -- "**/.env*"` in each repo to confirm no historical commits contain secrets.
