### 2. "Hackerbot-Claw" — Autonomous AI Bot Hits GitHub Actions at Scale
**Relevance: 10/10** | `#security` `#github` `#cicd` `#supplychain` `#agents`

An autonomous AI bot named "hackerbot-claw" systematically compromised CI/CD pipelines across 7+ major open-source repos (Microsoft, DataDog, Trivy and others), scanning 47,000 repositories in a week. Vector: malicious instruction injected into GitHub issue titles, picked up by AI triage bots ("Clinejection" attack pattern), triggering malicious Actions runs that exfiltrated secrets. OpenSSF has issued a formal warning. The payload domain was registered Feb 24, 2026.

**So what:** Our repos (AgentSteve80) use GitHub Actions. Priority: audit all workflow files for `pull_request_target` triggers with write permissions, pin Actions to commit SHAs (not tags), and don't let bots auto-run on untrusted input. Consider adding CodeQL scanning. The Clinejection pattern (issue title → AI bot → arbitrary code) is directly relevant if we ever wire an AI triage bot to our repos.

### 10. "Clinejection" — AI Triage Bot Supply Chain Attack Pattern (4,000 Machines Compromised)
**Relevance: 9/10** | `#security` `#agents` `#supplychain` `#cicd`

Detailed post-mortem on "Clinejection" — a 5-step supply chain attack that compromised ~4,000 developer machines in February 2026. An attacker injected a malicious instruction into a GitHub issue title; an AI triage bot reading the issue parsed it as a command and executed it, triggering a malicious workflow. The AI Security Digest also flags agent supply chain abuse scaling through skill marketplace scams and Unicode instruction injection.

**So what:** This is the exact threat model for any AI bot with repo access. Critical lesson: AI bots that process external input (issues, PRs, comments) must treat that input as untrusted. Never grant repo-write or secrets access to bots that ingest public user content. Relevant before we wire any AI automation to the Pal or SiteSignal repos.
